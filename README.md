# Beton Menu — refactor

Этот вариант разносит прежний монолит на слои.

## Что было изменено

- данные меню вынесены из `menu.html` в `src/data/menu-data.json`;
- `menu.html` стал тонкой оболочкой, в которой нет захардкоженных блюд и напитков;
- логика загрузки данных вынесена в `MenuRepository`;
- бизнес-логика навигации вынесена в `MenuService`;
- отрисовка DOM вынесена в `MenuRenderer`;
- обработка пользовательских действий вынесена в `MenuController`;
- кастомные HTML-теги вроде `<item>`, `<name>`, `<weight>`, `<price>` заменены на обычную разметку, которую проще поддерживать.

## Ответственность модулей

### `MenuRepository`
Отвечает только за загрузку данных.

### `MenuService`
Содержит доменные правила:
- валидация данных меню;
- выбор стартовой категории и стартового раздела;
- переключение категорий;
- переключение разделов;
- построение `viewModel` для UI.

### `MenuRenderer`
Ничего не знает о бизнес-правилах.
Он просто получает готовую модель и рисует интерфейс.

### `MenuController`
Связывает UI и доменную логику:
- слушает клики;
- меняет состояние;
- инициирует перерисовку.

## Как расширять дальше

- добавить поиск по меню в `MenuService`;
- вынести форматирование цен/объёмов в отдельные утилиты;
- добавить фильтры `isAvailable`, `tags`, `seasonal` в `menu-data.json`;
- заменить `fetch` на API без переписывания UI и доменной логики.
